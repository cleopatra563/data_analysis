# 用户人群标签的本质定义与区别

## 1. 核心定义与本质

### 1.1 活跃用户 (Active Users)
**本质**：在特定时间窗口内有过指定行为（如登录、使用核心功能、消费等）的用户，反映产品当前的实时用户基础和使用热度。

**判断条件**：
- 在观察周期内（通常是日、周、月）有过活跃行为
- 活跃行为的定义取决于业务场景（如登录、点击、购买等）

**业务意义**：
- 衡量产品当前的用户规模和健康度
- 评估运营活动的即时效果
- 是计算其他用户指标的基础

### 1.2 留存用户 (Retained Users)
**本质**：在两个连续或指定的时间周期内都有活跃行为的用户，反映产品对用户的持续吸引力和粘性。

**判断条件**：
- 在前一个时间周期（基准周期）内活跃
- 在后续的目标时间周期内仍然活跃

**常见类型**：
- 次日留存：基准日活跃，次日也活跃
- 7日留存：基准日活跃，第7日也活跃
- 30日留存：基准日活跃，第30日也活跃

**业务意义**：
- 评估产品的长期价值和用户粘性
- 衡量产品核心功能的吸引力
- 是产品迭代效果的重要指标

### 1.3 回流用户 (Returning Users)
**本质**：曾经活跃过但后来流失，在观察周期内重新活跃的用户，反映产品的召回能力和用户的回归意愿。

**判断条件**：
- 在观察周期内活跃
- 在前一个或多个连续时间周期内不活跃（流失状态）
- 再往前的时间周期内曾有过活跃记录

**业务意义**：
- 衡量流失用户的召回效果
- 评估产品新版本或召回活动的吸引力
- 反映产品的用户生命周期管理能力

### 1.4 流失用户 (Churned Users)
**本质**：曾经活跃过但在连续多个时间周期内都不活跃的用户，反映产品可能存在的问题和用户的不满。

**判断条件**：
- 在观察周期内不活跃
- 在前一个或多个时间周期内曾有过活跃记录
- 流失的定义通常是连续N个周期不活跃（如30天、60天等）

**业务意义**：
- 识别产品存在的问题和用户痛点
- 评估用户流失的风险和规模
- 为召回策略提供目标用户群体

## 2. 关系与区别

| 用户类型 | 核心特征 | 时间维度 | 业务关注点 |
|---------|---------|---------|-----------|
| 活跃用户 | 当前周期内活跃 | 单周期 | 实时用户规模 |
| 留存用户 | 连续周期内活跃 | 多周期（连续） | 用户粘性 |
| 回流用户 | 流失后重新活跃 | 多周期（间隔） | 召回效果 |
| 流失用户 | 连续周期不活跃 | 多周期（连续不活跃） | 用户流失风险 |

## 3. 计算逻辑关系

用户生命周期中的典型流转路径：
1. 新用户 → 活跃用户
2. 活跃用户 → 留存用户（持续活跃）
3. 活跃用户 → 流失用户（停止活跃）
4. 流失用户 → 回流用户（重新活跃）
5. 回流用户 → 留存用户 或 再次流失

## 4. 关键计算要点

- **时间窗口定义**：需明确日/周/月的具体划分方式（自然时间或滑动窗口）
- **活跃行为定义**：需根据业务场景定义有效的活跃行为（如登录、核心功能使用、消费等）
- **流失阈值**：需根据产品特性和用户行为习惯定义流失的时间阈值（如连续30天不活跃）
- **去重处理**：同一用户在同一时间周期内的多次活跃行为应只计算一次
